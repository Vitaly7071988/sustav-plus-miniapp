# Сустав+ AI-помощник — план подключения

В версии v0.3 внутри приложения уже есть экран **AI-помощник**.
Сейчас он работает в тестовом режиме:

- быстрые вопросы;
- локальные безопасные ответы;
- проверка красных флагов;
- fallback, если сервер не подключён.

## Как подключить настоящий OpenAI API позже

Нельзя вставлять OpenAI API key прямо в `app.js`, `index.html` или любой frontend-код. Ключ должен жить только на сервере.

Схема:

```text
Telegram Mini App → /api/assistant → OpenAI Responses API → ответ в Mini App
```

В папке уже есть заготовка:

```text
api/assistant.js
```

Она подходит для Vercel Serverless Function.

## Что нужно сделать на Vercel

1. Залить репозиторий на GitHub.
2. Подключить GitHub к Vercel.
3. В Vercel открыть проект → Settings → Environment Variables.
4. Добавить:

```text
OPENAI_API_KEY=ваш_ключ
OPENAI_MODEL=модель_из_вашего_аккаунта
```

5. Перезапустить деплой.
6. В приложении открыть вкладку **AI** и задать вопрос.

## Роль AI-помощника

Он НЕ должен:

- ставить диагнозы;
- назначать лекарства;
- отменять назначения врача;
- давать опасные советы;
- обещать результат;
- заменять хирурга, врача или реабилитолога.

Он должен:

- объяснять простыми словами;
- помогать сориентироваться;
- напоминать про красные флаги;
- предлагать снизить нагрузку при откатах;
- направлять к врачу при опасных симптомах;
- учитывать день программы, боль, воду и режим дня.

## Важная идея для будущей админки

Позже можно сделать кабинет администратора:

- список пользователей;
- последние вопросы;
- красные флаги;
- заявки на индивидуальный маршрут;
- источник врача/ref-ссылка;
- ручной ответ от Виталия;
- автоответ AI + отметка “нужно проверить человеку”.

Так безопаснее, чем полностью автономный “робот-врач”.
