# Сустав+ — аналитика v1.2

В этой версии приложение умеет отправлять события по пользователям и кликам.

## Что уже встроено

1. **Vercel Web Analytics** — базовые посетители и просмотры страниц.
2. **Свой трекер `/api/track`** — события кликов и действий внутри Mini App.
3. **Источник входа** — `startapp` / `ref`: `newsustav`, `youtube`, `site`, `vk`, `elkin`, `savinkov`, `karpovich`, `victoria`.

## Какие события отправляются

- `app_open` — человек открыл приложение;
- `screen_view` — открыл экран;
- `nav_click` — нажал кнопку/меню;
- `program_open` — открыл маршрут 14/30 дней;
- `day_open` — открыл день маршрута;
- `exercise_open` — открыл упражнение;
- `exercise_toggle` — отметил упражнение выполненным;
- `task_toggle` — отметил пункт дня;
- `routine_toggle` — отметил дыхание/микро-зарядку;
- `water_change` — добавил воду;
- `tracker_save` — сохранил трекер боли;
- `assistant_ask` — задал вопрос AI;
- `external_click` — перешёл по внешней ссылке.

## Минимальный просмотр прямо сейчас

В Vercel открой проект `sustav-plus-miniapp` → **Analytics**.
Там будут посетители и просмотры страниц после включения Web Analytics.

## Чтобы видеть клики в Telegram

В Vercel → Project → Settings → Environment Variables добавь:

```text
TELEGRAM_BOT_TOKEN=токен бота @sustavplus_app_bot
ANALYTICS_CHAT_ID=твой Telegram user_id или id закрытого чата/канала для логов
```

Опционально:

```text
ANALYTICS_TELEGRAM_EVENTS=all
```

Если `all` не ставить, в Telegram будут приходить только самые важные события: вход, открытие маршрута, упражнение, сохранение трекера, вопрос AI, внешний клик.

После добавления переменных нужно сделать **Redeploy** проекта в Vercel.

## Чтобы вести таблицу в Google Sheets

Смотри файл `docs/GOOGLE_SHEETS_ANALYTICS.md`.
В итоге в Vercel нужно добавить:

```text
ANALYTICS_WEBHOOK_URL=https://script.google.com/macros/s/....../exec
```

После этого события будут записываться в таблицу, где можно считать пользователей, клики, источники и популярные разделы.

## Важное по приватности

Не собираем диагнозы и медицинские данные специально. События нужны для продуктовой аналитики: кто зашёл, что открыл, где нажал, какой источник входа.
